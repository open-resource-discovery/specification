# ORD Standardized Concepts

The `ord` [vendor namespace](../../spec-v1/index.md#vendor-namespace) is **reserved** by the ORD specification itself.
It is used to define standardized [Concept IDs](../../spec-v1/index.md#concept-id) that have cross-vendor, specification-level semantics.

The `ord` namespace MUST NOT be used by ORD providers for their own content.

## Relation Types

Standardized relation types can be used in the `relationType` property of relationship objects
(e.g., `relatedApiResources`, `relatedEventResources`, `relatedEntityTypes`, `relatedCapabilities`).

| Concept ID | Description | Applicable To |
|---|---|---|
| `ord:sameAs` | The source and target resource are the same or equivalent. This is useful when the same resource is described by different ORD providers or in different contexts, and the relationship expresses that they refer to the same underlying resource. | All `relationType` properties |
